annotation		xml
@Autowired(setter)	property

@Component(@Controller, @Service, @Repository) : 객체화 (스프링의 BeanFactory라는 팩토리 패턴의 구현체에서 bean이라는 스프링프레임워크가 관리하는 객체가 있는데, 해당 클래스를 그러한 bean 객체로 두어 스프링 관리하에 두겠다는 어노테이션)

@RequestMapping("") : url 맵핑

@Autowired : 타입을 기반 (스프링프레임워크에서 관리하는 bean 객체와 같은 타입의 객체를 찾아서 자동으로 주입해주는 것. 해당 객체를 bean으로 등록하지 않으면 주입해줄 수 없다.)
@Resource : 이름을 기반
필드 앞에 위에 두개를 쓰면 기본생성자를 호출되면서 setter 사용

클라이언트가 서버에 데이터를 보내는 방법은 get과 post 두가지 방식
get : url에 표시됨
post : 표시x, html헤더를 통해 전송

@RequestBody : http 요청 몸체를 자바 객체로 변환 (http 요청 몸체를 자바 객체로 전달받을수 있음)
@ResponseBody : 자바 객체를 http 응답 몸체로 변환 (자바 객체를 http 응답 몸체로 전송할 수 있음)

@GetMapping = @RequestMapping(value="~", method=RequestMethod.GET)	get요청에만 반응
@PostMapping = ''
@RequestMapping : get, post 요청 둘다 반응
Post : Create : 리소스를 생성
GET : Read : 해당 URI의 리소스 조회
PUT : Update : 해당 URI의 리소스를 수정
DELETE : Delete : 해당 URI의 리소스를 삭제
